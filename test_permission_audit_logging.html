Instructions for Testing Permission Audit Logging

  1. Go to Administrator page ‚Üí Permissions tab
  2. Toggle any permission (e.g., Finance Executive ‚Üí Dashboard ‚Üí View)
  3. Wait 2-3 seconds for the audit log to be created
  4. Check the Audit Log section on the same page
  5. Look for "Permission Updated" entries
  6. Refresh this test page and click "Check Recent Audit Logs"

What to Look For:

  • Action: "Permission Updated" or "Updated Permissions"
  • Details: Should show role, module, action, and old ‚Üí new values
  • Entity Type: "Permission System" or "Permissions"
  • User: Your admin username
  • Timestamp: Recent time when you made the change

Troubleshooting

If audit logs are not appearing:

  1. Check browser console for error messages
  2. Verify backend is running on http://localhost:8000
  3. Check Network tab for failed API calls
  4. Try refreshing the Administrator page
  5. Check if you're logged in as Super Admin
